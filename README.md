# 企业采购、仓库、销售管理系统 V1.0

这是一个轻量级局域网网页系统，使用：

- 前端：HTML + CSS + JavaScript
- 后端：Python + Flask
- 数据库：SQLite

## 1. 安装依赖

在本项目目录打开终端：

```bash
python3 -m pip install -r requirements.txt
```

如果只是先试用，系统主要依赖 Flask；`openpyxl` 用于导入 Excel 文件。

## 2. 启动系统

```bash
python3 app.py
```

本机访问：

```text
http://127.0.0.1:8765
```

局域网其他电脑访问时，把 `127.0.0.1` 换成这台 Mac 的局域网 IP，例如：

```text
http://192.168.1.100:8765
```

## 3. 默认账号

管理员：

```text
用户名：admin
密码：admin123
```

普通用户：

```text
用户名：user
密码：user123
```

正式使用前，建议先用管理员账号新增自己的账号，然后修改默认密码。

## 4. 数据保存位置

数据库文件：

```text
data/app.db
```

上传附件：

```text
uploads/
```

供应商附件说明：

- 新增供应商时，`公司资料` 可以上传一个主要资料文件，例如营业执照、资质 PDF、报价 Excel。
- `照片` 用于上传供应商相关图片，例如门头、铭牌、现场照片。
- `多个附件` 支持一次选择多个文件，适合放合同、报价单、资质文件、对账资料等。
- 支持的格式：`jpg`、`jpeg`、`png`、`pdf`、`doc`、`docx`、`xls`、`xlsx`。
- 在供应商列表点击 `附件`，可以继续上传、下载或删除该供应商的附件。

手动备份：

```text
backups/
```

系统每天第一次被访问时，会在 `backups/` 里自动生成一份 `auto_backup_年月日.db`。

## 5. 简单测试流程

1. 登录管理员账号。
2. 新增一个供应商。
3. 新增一个客户。
4. 新增一个产品，填写安全库存和采购价、销售价。
5. 新增采购单，保存后查看库存是否增加。
6. 新增销售单，保存后查看库存是否减少。
7. 打开首页，看库存预警和本月采购、销售、利润。
8. 点击“备份数据库”，确认 `backups/` 目录出现备份文件。

## 6. 注意事项

- 删除数据只有管理员能操作。
- 普通用户可以查询、新增采购单、新增销售单。
- 销售单保存时会检查库存，库存不足会阻止保存。
- 数据库是系统核心文件，修改或升级前请先备份 `data/app.db`。
- 当前 PDF 导出以浏览器打印为主，可以在打印窗口选择“保存为 PDF”。

## 7. Excel 模板对应关系

项目里的 `管理系统.xlsx` 已作为当前字段模板参考：

- `采购` 工作表：供应商录入模板、采购订单模板。
- `仓库` 工作表：产品模板。
- `销售` 工作表：客户录入模板、销售订单模板。
- `统计` 工作表：按日、周、月、季度、年度统计，以及单品、单订单、单客户统计方向。

系统目前支持批量导入：

- 供应商：读取 `采购` 工作表里的 `供应商录入模版`
- 产品：读取 `仓库` 工作表里的 `产品模版`
- 客户：读取 `销售` 工作表里的 `客户录入模版`
- 采购订单：可以直接按采购订单模板导入，导入后会自动增加库存
- 销售订单：需要先有客户和产品，导入后会自动扣减库存

导入文件支持 `CSV` 和 `XLSX`。

如果导入 `XLSX`，系统会先找对应工作表和模板标题，再读取标题下一行的表头，所以可以直接按 `管理系统.xlsx` 的格式扩展数据。

如果导入 `CSV`，第一行需要使用表头，例如：

```text
编号、名称、等级、联系人、电话、地址、账户信息、公司类型、结算时间
备件编号、产品名称、技术描述、单位、当前库存、安全库存、应用设备、备件类型、质保时间、产地
应付金额、实收金额
```

推荐导入顺序：

```text
1. 供应商
2. 产品
3. 客户
4. 采购订单
5. 销售订单
```

采购订单导入说明：

- 采购订单 Excel 按 `管理系统.xlsx` 里的 `采购 → 采购订单版` 字段即可。
- 字段顺序是：`采购单号、供应商、采购日期、备件编号、产品名称、技术描述、单位、数量、单价、合计、照片、应用设备、备件类型、质保时间、产地`。
- `采购单号` 不填时，系统会自动生成；如果填写，不能和已有采购单号重复。
- `供应商` 可以填写供应商名称，也可以在软件导入区域选择供应商；如果都不填，系统会自动使用“未指定供应商”。
- `采购日期` 不填时，系统会自动使用当天日期。
- 如果产品台账里还没有对应备件编号，系统会根据采购订单里的产品信息自动建立产品，再增加库存。

库存管理导入说明：

- 库存管理页面支持导入产品 Excel。
- 对应 `管理系统.xlsx` 里的 `仓库 → 产品模版`。
- 字段顺序是：`备件编号、产品名称、技术描述、单位、当前库存、安全库存、照片、应用设备、备件类型、质保时间、产地`。
- 导入后会出现在库存管理的“实时库存”表里。

销售订单导入说明：

- 销售订单 Excel 按 `管理系统.xlsx` 里的 `销售 → 销售订单模版` 字段即可。
- 字段顺序是：`备件编号、产品名称、技术描述、单位、数量、单价、合计、货期`。
- 导入销售订单前，可以在软件导入区域选择客户；如果不选，系统会自动使用“未指定客户”。
- 销售导入会检查库存，库存不足的行会跳过并显示原因。

项目里已生成 5 个单独的示例 Excel 文件，路径是：

```text
导入示例Excel/
```

说明：

- 客户模板里的“应付金额”在系统里对应客户累计销售应付金额。
- 客户模板里的“实收金额”需要人工录入。
- 产品模板里的“当前库存”作为初始库存；正式使用后建议通过采购、销售、手工出入库来改变库存。
- Excel 里的“液体”已保留为备件类型选项，同时也保留“液压”，避免现场模板和原需求不一致时无法选择。
- Excel 里的采购订单和销售订单现在都支持导入；采购、销售导入后会自动更新库存流水。
